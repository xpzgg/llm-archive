CONTEXT · 2019 MAP, CURRENT MECHANISM

2019 年的“两级分派”地图仍然有用，但外层机制已经换代

CALL TIME · CONTEXT AS AN ORDERED KEYSET

Dispatcher 读取的是调用语境，而不只是 device

KeySet = Tensor Keys ∪ TLS Included − TLS Excluded ∩ Non-Fallthrough

Tensor Keys

对象携带的 backend、layout 与包装能力。

TLS Included

动态作用域主动开启的模式或默认职责。

TLS Excluded

已完成的层暂时屏蔽自己，避免重复处理。

Fallthrough

该算子在某一层无需工作，查找时直接跳过。

输入不是“CUDA Tensor”，而是一组有优先级的能力
当前 DispatchKeyExtractor 先从所有 Tensor 参数合并 key_set，再应用 TLS include/exclude 和该算子的 non-fallthrough mask。这里用交集表达 non-fallthrough。最终 OperatorEntry::lookup 由最高优先级计算表索引。

与 2019 年 Variable → Type 的单向模型相比，当前 Dispatcher 把多种语义统一为有序层。Dispatcher::redispatch 接收已经调整过的 KeySet，直接继续 lookup，不重新从原始 Tensor 计算。

AT_DISPATCH 选择 scalar_t

设计边界：dtype 不进入全局 Dispatcher 的组合空间

CONTIGUOUS

launch_vectorized_kernel
STRIDED

offset calculator
GENERAL

current CUDA stream
越接近硬件，算子身份越不重要；内存布局与并行策略越重要。
当前代码并非固定一个 elementwise_kernel，而是多种 launch policy
这里修正笔记里较理想化的固定 grid/block → elementwise_kernel 图。当前 CUDALoops.cuh 会根据连续性、32 位索引、向量宽度等走 vectorized、unrolled 或 offset-aware 路径，并使用当前 CUDA stream。
