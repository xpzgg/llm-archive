# 数的表示

## 原码、反码、补码

**核心问题**：计算机只有加法器，如何用加法模拟减法？

### 原码
最高位表示符号，其余位表示绝对值。直接用加法做减法结果错误，且0有两种表示。

### 反码
负数 = 原码各位取反（符号位不变）。加法接近正确，但需要"循环进位"修正，0仍有两种表示。

### 补码
负数 = 反码 + 1。本质是**模运算**：在模 2ⁿ 的世界里，-x 和 (2ⁿ - x) 是同一个数。

```
-5 的补码（4位）= 16 - 5 = 11 = 1011
5 + 11 = 16 ≡ 0 (mod 16) ✓
```

**为什么补码是最终答案**：
- 加法直接正确，无需修正
- 0 的表示唯一
- 比原码多表示一个数（如4位可表示 -8~+7）

现代所有计算机整数都用补码，加法器不需要知道操作数正负，统一处理。

## 定点数

**本质**：把小数问题转化为整数问题，通过约定小数点位置（隐式，不占存储位）来表示小数。

```
定点数真实值 = 存储的整数值 × 2^(-小数点后位数)
```

**小数点为什么是 2 的负次方**：不是约定，是位置计数法的数学延伸。第 n 位权重 = 基数^n，n 可以是负数，小数点只是 n=0 和 n=-1 之间的分界线。

**优点**：精度均匀、硬件简单、运算快  
**缺点**：动态范围窄

**应用**：金融计算（不能用浮点）、DSP、神经网络量化（INT8推理）

## 浮点数

**核心矛盾**：定点数精度和范围不可兼得，自然界的数跨越几十个数量级。

**设计精髓**：把精度和范围分开——尾数决定精度，指数决定范围（缩放比例），两者正交独立。

```
(-1)^S × 1.M × 2^(E-bias)
   ↑        ↑      ↑
  正负    有效数字  数量级
```

### IEEE 754 单精度（32位）

```
┌─┬────────┬───────────────────────┐
│S│ 指数 E │       尾数 M          │
│1│  8位   │        23位           │
└─┴────────┴───────────────────────┘
```

- **隐含前导1**：规格化数尾数永远是 1.xxx，1 不存储，23位实际有24位精度 ≈ 7位十进制有效数字
- **移码（偏置127）**：指数存储值 = 真实指数 + 127，使浮点数大小比较可直接当整数比较
- **特殊值**：全0=±0，指数全1尾数全0=±∞，指数全1尾数非0=NaN

### "浮"在哪里

容易误解：以为小数点在23位尾数内部移动。

实际：尾数内部结构固定，小数点永远在隐含1之后：`1.MMMMM...`，从不移动。

"浮"的是整个 `1.M` 被指数缩放后，落在数轴上的**位置**：
```
同样的尾数 1.01，指数不同 → 落在数轴的不同位置
1.01 × 2⁰  = 1.25
1.01 × 2¹⁰ = 1280
```

### 尾数精度是"间接"控制最终精度

尾数控制**相对精度**（有效数字位数恒定），乘以指数的缩放比例后才得到**绝对精度**。
两者缺一不可，尾数只能间接控制最终精度。

### 精度分布

浮点数在数轴上**不均匀分布**，靠近0密集，远离0稀疏。相对精度恒定（24位有效位），绝对精度随数量级变化。

```
最终绝对精度 = 初始精度(2^-23) × 放缩比例(2^(E-127))
```

这就是 `0.1 + 0.2 ≠ 0.3` 的原因：0.1 在二进制是无限循环小数，存储时截断产生误差。

### 单精度 vs 双精度

|  | 单精度 | 双精度 |
|---|---|---|
| 总位数 | 32 | 64 |
| 指数位 | 8 | 11 |
| 尾数位 | 23 | 52 |
| 有效数字 | ~7位 | ~15位 |
| 数值范围 | 10^±38 | 10^±308 |

### 浮点 vs 定点

| | 定点 | 浮点 |
|---|---|---|
| 精度分布 | 均匀 | 集中在小数附近 |
| 动态范围 | 窄 | 极宽 |
| 运算结合律 | 满足 | 不满足 |
| 适用 | 金融、DSP、AI推理 | 科学计算、通用计算 |
